เพื่อสะท้อนถึงระบบสถาปัตยกรรมเครือข่ายและการสร้างรายได้ (Monetization) ที่เพิ่มเข้ามา:
* **ภาษาไทย:** ระบบผู้เชี่ยวชาญเพื่อแนะนำการเลือกรถยนต์คันแรก พยากรณ์ความคุ้มค่า และ**ระบบจัดวางโฆษณาจำเพาะบุคคลด้วยเทคนิคการแคชบนเครือข่ายส่งมอบข้อมูล**
* **ภาษาอังกฤษ:** First-Car Recommendation Expert System, Cost-effectiveness Prediction, and **Low-Latency Personalized Ads Placement using CDN Caching**
### 2. หลักการและเหตุผล (Background & Rationale)
* **สิ่งที่ต้องเพิ่ม:** แพลตฟอร์มแนะนำรถยนต์ทั่วไปจำเป็นต้องมีโมเดลธุรกิจที่ยั่งยืน เช่น ระบบโฆษณา (Ads Platform) แต่การโหลดโฆษณาแบบดั้งเดิมที่ต้องวิ่งไปประมวลผลที่ฐานข้อมูลกลาง (Origin Server) ทุกครั้ง ส่งผลให้หน้าเว็บโหลดช้า (High Latency) และกินทรัพยากรระบบสูง โครงงานนี้จึงนำแนวคิด **Edge Computing / CDN Caching** มาใช้ร่วมกับผลลัพธ์การจัดกลุ่มพฤติกรรม (User Segmentation) เพื่อเสิร์ฟโฆษณาที่ตรงใจผู้ใช้ในระดับมิลลิวินาที (Low Latency)
### 3. วัตถุประสงค์ (Objectives)
* **เพิ่มข้อที่ 4:** เพื่อออกแบบและพัฒนาสถาปัตยกรรมการจัดวางโฆษณา (Ads Placement) ที่สามารถดึงข้อมูลโฆษณาตามความสนใจของกลุ่มผู้ใช้จาก CDN Cache ได้โดยไม่ต้องร้องขอไปยังเซิร์ฟเวอร์หลักทุกครั้ง
### 4. ขอบเขตและเครื่องมือ (Scope & Tech Stack)
จากเดิมที่มีเพียง Python, Scikit-learn และ Streamlit จำเป็นต้องเพิ่มโครงสร้างพื้นฐานด้าน Network/Edge:
* **CDN & Edge Platform:** ขยายขอบเขตโดยใช้ **Cloudflare Workers** (หรือ Vercel Edge Functions / AWS CloudFront Functions) เข้ามาช่วยตรวจจับ Request ที่ Edge Server
* **กลไกการแคช:** ใช้เทคนิค **Segment-based Caching** ผ่าน HTTP Header (เช่น Vary: X-User-Segment) หรือทำ **Edge-Side Includes (ESI)** เพื่อเย็บหน้าเว็บ (Stitch) ระหว่างเนื้อหาหลักที่แคชไว้กับโฆษณาที่เปลี่ยนไปตามกลุ่มผู้ใช้
## 🔄 แผนภาพการทำงานใหม่ (New Architecture Workflow)
นักศึกษาจะต้องปรับเปลี่ยน Flow ในวิธีการดำเนินงาน (Methodology) เล็กน้อย โดยเชื่อมโยงโมเดลเข้ากับระบบ Network ดังนี้:
```

[CDN Edge Server (เช่น Cloudflare)]
│───► (ดูว่ามี Cookie ระบุกลุ่ม "K-Means Cluster" ของคนนี้ไหม?)[span_7](start_span)[span_7](end_span)
│
├───► [มี] ──► ดึงโฆษณาที่แคชไว้สำหรับ Cluster นั้นๆ เสิร์ฟให้ทันที (0-10ms)
│
└───► [ไม่มี] ──► ส่งต่อไปที่ [Streamlit Application (Origin Server)][span_8](start_span)[span_8](end_span)
│
▼
[K-Means Predict][span_9](start_span)[span_9](end_span) ──► ฝัง Cookie "Segment-A" กลับไปที่บราวเซอร์

```
1. **User Segmentation:** เมื่อผู้ใช้ทำแบบทดสอบพฤติกรรม ระบบจะใช้ **K-Means** จัดกลุ่มผู้ใช้ (เช่น กลุ่มงบประหยัดเน้น Eco-car, กลุ่มเทคโนโลยีเน้น EV)
2. **Tagging:** ระบบจะฝัง Cookie หรือ JWT Token ระบุกลุ่ม (Segment) ไว้ที่ฝั่งผู้ใช้งาน
3. **CDN Routing:** ในการเข้าชมครั้งต่อไป CDN Server ที่อยู่ใกล้ตัวผู้ใช้ที่สุด จะอ่านค่า Cookie นี้ แล้วดึงแบนเนอร์โฆษณาที่แมตช์กับกลุ่มนั้น (เช่น โฆษณาตู้ชาร์จไฟสำหรับกลุ่ม EV หรือดีลประกันภัยราคาถูกสำหรับกลุ่ม Eco-car) จาก Cache ออกมาแสดงผลทันทีโดยไม่ต้องไปกวนใจ Python Backend
